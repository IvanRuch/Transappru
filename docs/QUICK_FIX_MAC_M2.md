# ⚡ Быстрое решение проблемы с TEMP на Mac M2

## Проблема
При запуске `npm run android` возникает ошибка:
```
EACCES: permission denied, mkdir '/var/folders/...'
```

## ✅ Решение (3 команды)

```bash
# 1. Создать свою временную папку
mkdir -p ~/tmp

# 2. Добавить переменные окружения
cat >> ~/.zshrc << 'EOF'
export TMPDIR=~/tmp
export TEMP=~/tmp
export TMP=~/tmp
EOF

# 3. Применить изменения
source ~/.zshrc
```

## 🚀 Запустить проект

```bash
# Перезапустить терминал или выполнить
source ~/.zshrc

# Запустить проект
npx expo start
npm run android
```

## 🔍 Проверка

После настройки проверьте:

```bash
# Должно показать: /Users/<username>/tmp
echo $TMPDIR

# Должна существовать
ls ~/tmp
```

Если видите путь к `~/tmp` - всё настроено правильно! ✅

---

## 💡 Почему это происходит?

Mac M2 (Apple Silicon) имеет более строгие права доступа к системным папкам `/var/folders/`. 
Решение - использовать свою временную папку в домашней директории.

---

## 📞 Всё ещё не работает?

1. Перезапустите терминал полностью (закройте и откройте заново)
2. Проверьте что переменные применились: `echo $TMPDIR`
3. Очистите кэш: `rm -rf node_modules && npm install`
4. См. полную документацию: `docs/SETUP_MAC_M2.md`
