# Логика выбора зон (МКАД, ТТК, СК)

## Обзор

В приложении реализована гибкая система выбора зон для заказа пропусков. Пользователь может выбирать зону вручную или система автоматически определяет зону на основе адреса, выбранного на карте.

## Зоны

- **МКАД** — Московская кольцевая автомобильная дорога
- **ТТК** — Третье транспортное кольцо
- **СК** — Садовое кольцо

## Логика работы

### 1. Автоматическое определение зоны

Когда пользователь выбирает адрес на карте (`PassYaMapScreen`):
- Система автоматически определяет зону на основе координат
- Зона передается обратно в `PassScreen`
- Флаг `isLocationTypeManual = false` (зона определена автоматически)

### 2. Ручной выбор зоны

Пользователь может вручную изменить зону на экране `PassScreen`:
- Нажатие на кнопку зоны (МКАД/ТТК/СК) изменяет выбор
- Флаг `isLocationTypeManual = true` (зона выбрана вручную)
- **Ручное изменение разрешено даже при выбранном адресе с карты**

### 3. Предупреждение при ручном изменении

Если пользователь вручную меняет зону при уже выбранном адресе с карты:
- Отображается предупреждающее сообщение желтого цвета
- Текст: "⚠️ Зона изменена вручную. Убедитесь, что выбранная зона соответствует адресу."
- Цель: информировать пользователя о потенциальном несоответствии

## Состояния компонента PassScreen

```typescript
interface PassState {
  location_type: string;        // 'mkad' | 'ttk' | 'sk' | ''
  lon: number | string;         // Долгота (с карты)
  lat: number | string;         // Широта (с карты)
  address: string;              // Адрес
  isLocationTypeManual: boolean; // Флаг ручного выбора зоны
}
```

## Сценарии использования

### Сценарий 1: Выбор адреса на карте
1. Пользователь нажимает "Выбрать на карте"
2. Выбирает точку на карте
3. Система определяет зону автоматически (например, ТТК)
4. Возврат на экран заказа → зона ТТК выбрана, `isLocationTypeManual = false`

### Сценарий 2: Ручное изменение зоны после карты
1. Пользователь выбрал адрес на карте (зона ТТК)
2. Вручную меняет зону на МКАД
3. Появляется предупреждение: "⚠️ Зона изменена вручную..."
4. `isLocationTypeManual = true`
5. При заказе используется МКАД (выбор пользователя)

### Сценарий 3: Ручной ввод адреса + выбор зоны
1. Пользователь вводит адрес вручную (без карты)
2. Выбирает зону МКАД
3. `isLocationTypeManual = true`
4. Предупреждение не показывается (нет координат)

### Сценарий 4: Очистка адреса
1. Пользователь очищает адрес
2. Координаты `lon` и `lat` сбрасываются
3. `isLocationTypeManual = false`
4. Зона остается выбранной, но предупреждение исчезает

## Визуальные индикаторы

### Кнопки зон
- **Выбранная зона**: серый фон `#D7D7D7`
- **Невыбранная зона**: светло-серый фон `#EEEEEE`

### Предупреждающее сообщение
- **Фон**: `#FFF3CD` (светло-желтый)
- **Рамка**: `#FFE69C` (желтый)
- **Текст**: `#856404` (темно-коричневый)
- **Иконка**: ⚠️

## Код

### PassScreen.tsx

```typescript
checkTab = (tab: string) => {
  const hasMapAddress = this.state.lon !== '' && this.state.lat !== '';

  if(this.state.location_type === tab) {
    // Снимаем выбор зоны
    this.setState({ location_type: '', isLocationTypeManual: false })
  } else {
    // Выбираем зону ВРУЧНУЮ
    this.setState({ 
      location_type: tab, 
      isLocationTypeManual: hasMapAddress ? true : true 
    })
  }
}
```

### Предупреждающее сообщение

```tsx

  }}>
    <Text style={{ fontSize: 12, color: '#856404', textAlign: 'center' }}>

    </Text>

```

## Причины изменения

**Старая логика (до изменений):**
- Зона блокировалась при выборе адреса на карте
- Пользователь не мог изменить зону, если привязка была неактуальной

**Новая логика (текущая):**
- Пользователь имеет полную свободу выбора зоны
- Система предупреждает о ручном изменении
- Баланс между автоматизацией и контролем пользователя
