- **2026-04-22**: UX: optimistic swap on web org switch. Root cause: backend serialises `/get-auto-list` per session, so the sidebar's fire-and-forget refetch got queued behind `AutoListScreen.loadData()` (full list) + `updateUserDataOnly` ‚ÄĒ user observed the sidebar keep showing the previous org for several seconds after the fleet already rendered. Fix: `WebSidebar.switchOrg` now updates local state the instant `/set-current-inn` succeeds ‚ÄĒ the clicked org moves into `userData`, the previously-current org takes its slot in `otherUserList`, the user's phone stays (session-bound). The background `loadData()` still runs with latest-wins AbortController semantics and reconciles any drift. Effect: sidebar footer + list reshuffle in ~500ms (round-trip of `/set-current-inn` alone), regardless of how long the full fleet refetch takes.
- **2026-04-22**: Upgrade `WebSidebar.loadData` from `inFlightRef` dedup to `AbortController` with **latest-wins** semantics. Reason: dedup drops new triggers while an old request hangs ‚ÄĒ meaning a stalled backend (ivan.trans-konsalt.ru sometimes takes >30s ‚Üí `AxiosError ECONNABORTED`) blocks the user for the full axios timeout before they can retry. With AbortController, a new trigger cancels the stale request immediately and pushes a fresh one; `axios.isCancel(e)` filter in the catch block swallows the expected cancellation noise. The controller is also aborted on unmount ‚Üí no `setState on unmounted component` warnings, no wasted bandwidth. Also discovered: web and mobile hit different backends (`api.ts` ‚Üí `transapp.ru`, `api.web.ts` ‚Üí `ivan.trans-konsalt.ru`) ‚ÄĒ documented as a possible cause of inter-platform data discrepancies during manual QA.
