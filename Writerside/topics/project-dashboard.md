Поэтому свой comparator. `localeCompare('ru')` остался для secondary tie-break, корректно работает с кириллицей. **Edge cases (по дизайну, не баги)**: пустой/undefined plate → primary 0 (попадёт в начало), нестандартный plate без цифр → 0, спецтехника типа `АА1234` → берёт первую digit-run = 1234, прицепы `34АВ` → 34. Соответствует user intuition «по цифрам в номере». **Pagination caveat**: backend всё ещё страничит лексикографически, при load-more окно может слегка сдвинуться — для типичных автопарков (< 100 авто на организацию) невидимо, при необходимости фикс уйдёт на backend через `sort_by=plate_digits`. **Tests**: новый файл `src/utils/__tests__/plateHelpers-sort.test.ts` — 11 кейсов (`plateSortKey` extraction для standard plates / non-digit / empty / non-standard layouts; `compareByPlateNumber` для numeric ordering / region-irrelevance (`А999БВ01` vs `А001БВ77` → 001 first) / fallback на `auto_number` без `auto_number_base` / alphabetical tie-break (`К123ОР99` vs `А123БВ77` → А first); `sortAutoListByPlateNumber` для empty / immutability / idempotency).
