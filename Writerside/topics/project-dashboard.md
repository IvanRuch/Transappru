- **2026-04-29**: Web favicon восстановлен + полноценный `<head>` + nginx security headers. Сделано в одном PR (`feat/web-head-and-security`): (1) **Иконка**: `transappweb/public/img/favicon.png` (66×66, единственная сохранившаяся легаси-иконка) пересэмплирован через `sips -Z` macOS в 5 стандартных размеров — `public/favicon-{16,32}.png`, `public/apple-touch-icon.png` (180×180), `public/icon-{192,512}.png`, плюс копия 32×32 как `public/favicon.ico` для legacy запросов; та же 32×32 версия положена в `assets/images/favicon.png` как backup. (2) **Кастомизация `<head>`**: создан `app/+html.tsx` — стандартная точка расширения Expo Router 6 для root HTML template (нативная поддержка в SDK 54, не требует `expo-head` package), рендерится один раз на route при `web.output: "static"`. Содержит: `lang="ru"`, `<title>`, description (`"Сервис для перевозчиков: проверка штрафов ГИБДД и РНИС, оплата через систему «Казна»..."`), robots, theme-color `#000000`, четыре `apple-mobile-web-app-*` метатега для iOS standalone, OG-набор (type/locale/site_name/title/description/image), Twitter Card (`summary`), 5 `<link rel="icon|shortcut icon|apple-touch-icon|manifest">`, и `<ScrollViewStyleReset/>` из `expo-router/html` (нужно для корректного RNWeb scroll). `web.favicon` из `app.json` удалён, чтобы Expo не дублировал свой `<link rel="icon">` поверх наших. (3) **PWA-минимум**: создан `public/manifest.webmanifest` (name/short_name TransApp, theme_color, background_color, lang ru-RU, display standalone, scope `/`, два icons 192/512 с purpose `any maskable`); `public/robots.txt` пермиссивный с пометкой «sitemap добавим когда появятся публичные landing-страницы». Expo Router 6 копирует `public/` в `dist/` нативно — никакого plugin-а не нужно, просто `COPY public/ ./public/` добавлен в `nginx/Dockerfile.prod` builder stage. (4) **Nginx security headers**: набор вынесен в shared `nginx/security-headers.partial.conf` (X-Frame-Options SAMEORIGIN, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin, Permissions-Policy с отключённой geolocation/mic/camera + payment=(self), и пр_прагматичная Content-Security-Policy с `unsafe-inline`+`unsafe-eval` в script-src — обязательно для Expo runtime/Metro bundle init и NativeWind inline styles), `include`-нут в каждый `location` с собственными `add_header` директивами. **Важная nginx-семантика**: `add_header` НЕ наследуется через nested блоки — любой `add_header` в дочернем `location` молча сбрасывает все унаследованные → отсюда необходимость явного include в /static/, /favicon.ico, регексп `/(favicon-(?:16|32)|apple-touch-icon|icon-(?:192|512))\.png$/`, /manifest.webmanifest, /robots.txt, /health, /index.html. Те же include-ы продублированы в HTTPS heredoc-конфиге внутри `nginx/docker/entrypoint.sh` (HSTS остаётся как был — он HTTPS-only). Новые location-блоки для favicon/icon-*.png дают year-cache `public, immutable`, а manifest/robots — короткий 1h cache + правильные MIME (`application/manifest+json`, `text/plain`). (5) **CSP rationale**: прагматичный набор с `'unsafe-inline'`/`'unsafe-eval'` в script-src — это compromise, потому что Expo runtime использует Function-constructor в React reconciler, а NativeWind инжектит inline стили; сужение до nonce-based CSP потребует ухода Expo Web с `'unsafe-eval'`, что нетривиально. `frame-ancestors 'none'` — современный strict-эквивалент `X-Frame-Options`. `connect-src` ограничен текущими staging endpoint-ами + Yandex Maps tiles + Kazna sandbox + `wss:` для будущих websocket-ов. **Verified end-to-end**: `npx tsc --noEmit` 0 errors, `npm test` 12/61 ✅, `docker build -f nginx/Dockerfile.prod` сборка за ~14 секунд (Expo Web export внутри изолированного контейнера, обходит конфликт Metro state с локальным `expo start`); smoke-test через временный контейнер на `payment-service_default` Docker network подтвердил: `/` отдаёт все 5 security headers + CSP, head содержит title/description/OG/Twitter/icon links/manifest, `/favicon.ico` 200 image/x-icon year-cache, `/manifest.webmanifest` 200 `application/manifest+json` 1h-cache, `/favicon-32.png`+`/apple-touch-icon.png` 200 image/png year-cache, `/robots.txt` 200 text/plain. **Documentation**: `Writerside/topics/dev-web.md` расширен новым разделом "Web `<head>` & static assets" + "Nginx security headers" с таблицей назначения каждого хедера, CSP rationale, командой regenerate-иконок через `sips`, и шагом локального smoke-теста. **Open**: `sitemap.xml` отложен — у staging-сайта одна публичная страница (auth-gate), индексировать пока нечего, добавим когда появятся public landing pages.
